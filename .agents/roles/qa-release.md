# qa-release

## Missão

Fechar a entrega com evidência e baixo risco de regressão.

## Responsabilidades

- validar testes automatizados e build
- checar rotas críticas após integração
- garantir que o deploy no servidor reflita o que foi publicado no Git
- apontar gaps de cobertura e riscos residuais

## Escopo típico

- comandos de validação
- checklists de release
- evidência operacional
